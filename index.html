What is Aether? ↓
01How it works

An AI builds the game live

Aether is a 3D multiplayer arena where the game master isn't a script — it's an AI that builds the game live around you.

🔮

The Aetherist composes

An autonomous agent picks templates, spawns creatures, casts spells, and starts mini-games — all in real time, reacting to who's playing and what they say in chat.

🎮

Six game modes, seventeen arenas

King of the Hill. Hot Potato. Race. Reach the Goal. Collect. Survival. Each mini-game spins up in seconds with physics-altering spells thrown in for chaos.

💬

Talk to the agent

Tag @agent in chat — "spawn a giant spider", "make the floor lava", "flip gravity". The Aetherist hears you and decides if your wish becomes reality.

02Built on 0G

A game master that lives on the AI Layer-1

Most "AI in a game" projects ship an LLM in a box. Aether is different — every layer touches 0G, end to end.

0G · Chain

One wallet for humans and AI alike

Player bribes settle as native 0G transfers on Galileo testnet. Verified server-side by viem against on-chain state — no oracles, no off-chain trust.

0G · Compute

The Aetherist thinks on-chain

The agent's reasoning runs on 0G's decentralized inference network. Every prompt is a signed request through the 0G serving broker; the Aetherist pays for its own compute from the same wallet that holds bribes.

0G · Storage

Replays + memory, addressable forever

Every game result and agent memory snapshot uploads to 0G Storage, returning a rootHash that anyone can reference. Sessions don't die — they get archived.
